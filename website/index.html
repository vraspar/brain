How it works

1. Create
$ brain init --name "Acme Engineering"
✅ Brain "Acme Engineering" is ready! Local: ~/.brain/repo Remote: https://github.com/acme/brain-hub.git Author: alice
2. Share
$ brain push --title "Docker Guide" \
--file ./guide.md
✅ Pushed: Docker Guide Tags: docker (auto-detected)
3. Discover
$ brain digest

Why brain?

Git as storage

Entries are markdown in a git repo. No server. No database. No infrastructure to manage. Version history and access control are free.

FTS5 search

SQLite FTS5 with BM25 ranking. Sub-millisecond search. Works offline. The index is a disposable cache — rebuilt from source on every sync.

MCP-native

Your AI agent already knows what your team knows. 5 tools and 2 resources exposed via Model Context Protocol. brain serve starts the server.

Zero infrastructure. One command to start.
brain connect https://github.com/acme/brain-hub.git

Works with your AI agent

Add to your MCP client config:

mcp.json
{ "mcpServers": { "brain": { "command": "brain", "args": ["serve"] } } }

5 tools: push · search · whats_new · get_entry · stats
2 resources: brain://digest · brain://stats

Your agent can search team knowledge, publish findings, and check what's new — without you asking it to.

Quick start

# Install
$ git clone https://github.com/vraspar/brain.git
$ cd brain && npm install && npm run build
$ npm link
 
# Create a brain (first person)
$ brain init --name "Acme Engineering"
 
# Or join an existing one (teammates)
$ brain connect <url>

Requires Node.js 18+ and git.
